Problem

Reporter adapters cross the seam and depend on TestContext/TestNode shape, so the interface is almost as complex as the implementation.

Solution

Concentrate lifecycle ordering and reportable suite facts in one deep reporting module, then keep output adapters thin.

Benefits

  • locality: reporter bugs concentrate
  • leverage: four adapters benefit
  • tests hit one interface
  • TestNode stops leaking

Problem

The transform modules duplicate class-member traversal and generate magic methods that runtime reflection must know implicitly.

Solution

Concentrate class-member planning in one transform module and keep generated behaviours behind that implementation.

Benefits

  • locality: transform rules concentrate
  • leverage: two methods share plan
  • README can name seam
  • runtime magic documented

Top recommendation

Start with Deepen the reporting lifecycle.

It has a real seam already: Summary, Verbose, CSV, JSON, and Combination are multiple adapters. Deepening it gives immediate locality, protects external reporters, and creates a smaller test surface before touching the larger TestContext and CLI modules.
